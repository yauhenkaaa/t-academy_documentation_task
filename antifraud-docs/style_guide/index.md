Этот раздел описывает правила оформления документации в проекте. Единый стиль делает документацию предсказуемой: читатель знает, где искать информацию, а автор — как её подавать.

:::tip Главное правило
Документация пишется для читателя, а не для автора. Если абзац понятен вам, но не понятен коллеге — его нужно переписать.
:::

## Зачем нужен style guide

Без договорённостей в команде документация разрастается непоследовательно:

- одни используют `##` там, где нужен `###`
- заголовки файлов не совпадают с `title` в front matter
- диаграммы есть, а текстового описания к ним нет

Style guide — не свод запретов, а набор решений, принятых один раз, чтобы не принимать их снова каждый раз при написании страницы.
